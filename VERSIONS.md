## v0.1.0 — 量化交易核心 🔧

**目标：** 策略管理 + 回测引擎 + 信号决策 + OKX 交易执行

### 技术选型

| 层 | 技术 | 说明 |
|----|------|------|
| 数据源 | OKX Agent Trade Kit MCP | 实时行情、K线、订单簿、70+ 指标 |
| 历史数据 | SQLite / Clickhouse | 定时拉取并存储 K 线数据用于回测 |
| 策略框架 | 后端加载 Python 策略文件 | 用户编写或选择策略类，动态加载 |
| 回测引擎 | backtrader | Python 最成熟的开源回测框架 |
| 前端图表 | ECharts / TradingView Widget | K 线 + 指标叠加 + 交易信号标注 |
| 执行层 | OKX Agent Trade Kit MCP | 模拟盘（`--profile demo`）→ 实盘（`--profile live`）|

### 子模块

#### A. 市场数据管线

| # | 功能 | 说明 |
|---|------|------|
| A1 | 行情数据拉取 | OKX MCP 定时拉取指定交易对的 K 线、深度、资金费率 |
| A2 | 历史数据存储 | SQLite 存 K 线、tick 级深度快照，供回调查询 |
| A3 | 实时行情面板 | 前端展示多个交易对的实时价格、涨跌幅、成交量 |

#### B. 策略管理

| # | 功能 | 说明 |
|---|------|------|
| B1 | 策略模板库 | 内置经典策略模板：双均线、MACD、网格、海龟交易 |
| B2 | 策略编辑器 | Monaco Editor 在线编写 Python 策略文件 |
| B3 | 策略参数配置 | 前端表单生成策略参数（指标周期、阈值等）|
| B4 | 策略版本管理 | 每次修改自动存储历史版本，可回退 |

#### C. 回测引擎

| # | 功能 | 说明 |
|---|------|------|
| C1 | 回测执行 | 选择策略 + 交易对 + 时间范围 → backtrader 跑回测 |
| C2 | 回测报告 | 总收益率、年化收益、夏普比率、最大回撤、胜率 |
| C3 | 回测可视化 | 资金曲线图 + K 线叠加买卖点标注 |
| C4 | 参数优化 | 网格搜索 / 遗传算法 优化策略参数组合 |

#### D. 信号决策层

| # | 功能 | 说明 |
|---|------|------|
| D1 | 信号生成 | 策略实时运行输出买卖信号（方向/价位/数量）|
| D2 | 风控规则 | 单笔最大亏损、单日最大交易次数、仓位限制 |
| D3 | 信号审核 | 人工确认模式：AI 生成信号 → 用户审核 → 执行 |
| D4 | 信号日志 | 所有信号完整记录，包含拒绝原因和执行结果 |

#### E. OKX 交易执行

| # | 功能 | 说明 |
|---|------|------|
| E1 | MCP 集成 | 将 `okx-trade-mcp` 注册到 Claude Code 配置 |
| E2 | 模拟盘 | `--profile demo` 模式下验证完整交易链路 |
| E3 | 实盘切换 | 通过配置文件切换 demo/live，前端显示当前模式 |
| E4 | 订单管理 | 前端查看当前持仓、挂单、历史订单 |

## v0.2.0 — 脚本管理中心 📋

**目标：** 通用的 Python 脚本管理 + 执行 + 定时调度。

| # | 功能 | 说明 |
|---|------|------|
| 1 | 脚本目录浏览 | 读取 `scripts/` 目录，树形展示所有 `.py` 文件 |
| 2 | 脚本在线编辑 | Monaco Editor 在线修改脚本 |
| 3 | 一键执行 | 前端触发 → 后端 subprocess 执行 → WebSocket 实时返回输出 |
| 4 | 执行历史 | 记录每次执行的时间、耗时、输出日志 |
| 5 | 脚本分类标签 | 用户可为脚本添加标签（文件处理 / 数据采集 / 量化）|
| 6 | 定时任务 | 基于 APScheduler 设置脚本 crontab 定时执行 |
